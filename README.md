# Dress Redesign Agent

An AI-powered tool that redesigns a dress from a photo. Upload an image, fill in your style preferences, and receive a validated fashion sketch + professional seamstress instructions — all generated by an 8-step multi-agent pipeline built on Google ADK and Gemini.

## What it does

1. **Trend Research** — searches Google for current fashion trends matching the chosen style
2. **Dress Analysis** — analyzes the uploaded photo using Gemini multimodal vision
3. **Design Creation** — generates a redesign concept grounded in trend research and user preferences
4. **Design Validation** — runs 6 structured checks for consistency, feasibility, and completeness before any image is generated
5. **Sketch Generation** — produces a front + back fashion sketch using Gemini image generation
6. **Sketch Validation** — visually compares the sketch against the design description using Gemini Vision; validates the sketch for up to three total attempts and regenerates it up to two times
7. **Seamstress Guide** — writes detailed professional construction instructions tied to the specific design
8. **Seamstress Validation** — checks the guide for both design consistency and professional construction quality; auto-fixes issues in a loop up to 3 times

## Architecture

```
SequentialAgent (DressRedesignPipeline)
├── TrendResearchAgent        — google_search tool, outputs trend_insights
├── DressAnalyzerAgent        — native multimodal, outputs dress_analysis
├── DesignCreatorAgent        — outputs design_concept
├── DesignValidatorAgent      — 6-check validation, outputs final_design_concept
├── ImageGeneratorAgent       — calls generate_dress_sketch tool
├── SketchValidatorAgent      — visual validation loop (up to 3 attempts)
├── SeamstressGuideAgent      — outputs seamstress_guide
└── SeamstressValidatorAgent  — quality validation loop (up to 3 attempts)
```

Each agent reads from and writes to a shared session state. Agents use `include_contents="none"` to prevent conversation history leakage between steps. Each pipeline run gets a unique UUID session and sketch filename for multi-user safety.

## Validation design

The pipeline uses a **generate → validate → fix** pattern at two stages:

- **SketchValidatorAgent** reads the generated sketch image and the `IMAGE_PROMPT` from the design concept, identifies missing or incorrect visual elements, and regenerates the sketch with explicit corrections until it passes or reaches 3 attempts.
- **SeamstressValidatorAgent** checks the guide against the design concept and user preferences, and also validates professional construction quality — correct technique names, construction sequence, grain direction, needle types, thread selection, and irreversible step marking. Rewrites problematic sections in a loop until approved.

## Tech stack

- [Google ADK](https://google.github.io/adk-docs/) 2.3.0 — multi-agent orchestration
- [Gemini 2.5 Flash](https://deepmind.google/technologies/gemini/) — design validation, sketch validation, seamstress validation
- [Gemini Flash Lite](https://deepmind.google/technologies/gemini/) — trend research, analysis, design creation, seamstress guide
- [Gemini 3.1 Flash Image](https://deepmind.google/technologies/gemini/) — sketch generation
- [Streamlit](https://streamlit.io/) — web UI

## Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Create a `.env` file in the project root:

```
GOOGLE_API_KEY=your_api_key_here
```

## Run

```bash
streamlit run app.py
```

## Test

```bash
pip install -r requirements-dev.txt
pytest tests/ -v
```

11 unit tests covering all 3 tools — no real API calls required (fully mocked).
